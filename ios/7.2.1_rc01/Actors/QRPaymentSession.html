  • generateQRPaymentData(withQRPaymentInputData:digitalCardID:userPrompt:) Asynchronous

    Generates QR payment data for QR payment transactions

    Throws:

    • qrPaymentFailed(_:) if QR payment has failed.
    • deviceEnvironmentUnsafe(_:) if device environment is unsafe.
    • unsupportedPaymentType(_:) if payment type is unsupported.
    • sessionInProgress if error encountered when one payment session is already in progress and trying to make another payment.
    • invalidQRInputData(_:) if the QR input data is invalid.
