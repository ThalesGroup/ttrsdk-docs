  • DigitalizedCardProfileKeys

    DigitalizedCardProfileKeys class @discussion This contains a list of keys for insensitive and custom data in the card profile that can be retrieved by the UI application, via API [DigitalizedCardDetails getValueWithKey:]. The data available here are classified to be insensitive and useful for End-User display if UI application desires to do so.

    Constants

    > The key to get the token ID associated with the card profile. Linked to the dc.tokenID field in the JSON structure. <br/> <br/>


```SCHEME```
> The key to get the textual representation of card scheme. The texts are coming from server, typically in block letters such as "MASTERCARD" or "VISA". <br/> <br/>


```CVM_RESET_TIMEOUT```
> The key to get the duration during which the CardHolder verification, once successfully verified, status remains valid.<br/> <br/>


```DUAL_TAP_TIMEOUT```
>   The key to get the maximum duration allowed between the first tap and second tap for the high-value transaction use case, whereas the first tap
    would trigger the UI application to perform CardHolder verification within this duration only.<br/> <br/>

```ADDITIONAL_DATA```
>   The key to get the additional and custom data. These data are not really part of the profile data, either added by the server or from the issuers. NOTE that the value of this key is in fact always a JSON dictionary object that contains key-value pairs. The server always inject in this value 2 mandatory keys, which are ADDITIONAL_DATA_PRODUCT_ID and ADDITIONAL_DATA_PAN_LAST_4_DIGITS.<br/>
An example of extracting additional data is as following:
<pre>
<code>
[card getCardDetails:^(id&lt;DigitalizedCardDetails&gt; cardDetails, NSError *error) {
    if(!error){
        NSString * additionalDataStr = [cardDetails getValueWithKey:ADDITIONAL_DATA];
        NSDictionary *additionalDataJson = [NSJSONSerialization JSONObjectWithData:[additionalDataStr dataUsingEncoding:NSUTF8StringEncoding] options:NSJSONReadingMutableContainers error:nil];

        // Get Product ID & Last 4 Digits
        NSString * productId = [additionalDataJson objectForKey:ADDITIONAL_DATA_PRODUCT_ID];
        NSString * last4Digits = [additionalDataJson objectForKey:ADDITIONAL_DATA_PAN_LAST_4_DIGITS];

        // Get some other custom data object, for example "custom_data"
        NSDictionary * customData = [additionalDataJson objectForKey:@"custom_data"];

        // If the data is nested, for example "custom_data" contains a key called "contract_id"
        NSString * contractId = [customData objectForKey:@"contract_id"];
    }
}];
</code>
</pre><br/> <br/>


```ADDITIONAL_DATA_PRODUCT_ID```
>   The key to get the Product ID of the card. Product ID is the standardized identification of the card, that is provisioned by the server. One of the common usages of this value is to allow mapping a given card with custom data in the UI application. For example, the card image is not provisioned to the SDK, but rather acquired through different channel by the UI application. The UI application would then be able to display the card list with the images correctly via this value.<br/>
An example of mapping a card to its image using Product ID is as following:
<pre>
<code>
[card getCardDetails:^(id&lt;DigitalizedCardDetails&gt; cardDetails, NSError *error) {
    if(!error){
        NSString * additionalDataStr = [cardDetails getValueWithKey:ADDITIONAL_DATA];
        NSDictionary *additionalDataJson = [NSJSONSerialization JSONObjectWithData:[additionalDataStr dataUsingEncoding:NSUTF8StringEncoding] options:NSJSONReadingMutableContainers error:nil];
        NSString * productId = [additionalDataJson objectForKey:ADDITIONAL_DATA_PRODUCT_ID];
        NSString * cardImgFileName = [NSString stringWithFormat:@"%@%@.png", BASE_PATH , productId];
       // Here calls loading card image and display
    }
}];
</code>
</pre><br/> <br/>


```ADDITIONAL_DATA_PAN_LAST_4_DIGITS```
>   The key to get the PAN last 4 digits of the card. Similar to ADDITIONAL_DATA_PRODUCT_ID, this value is typically used to identify the card, either by the UI application or by the End-User. <br/>
An example of mapping a card to its image using PAN last 4 digits is as following

<pre>
<code>
[card getCardDetails:^(id&lt;DigitalizedCardDetails&gt; cardDetails, NSError *error) {
    if(!error){
        NSString * additionalDataStr = [cardDetails getValueWithKey:ADDITIONAL_DATA];
        NSDictionary *additionalDataJson = [NSJSONSerialization JSONObjectWithData:[additionalDataStr dataUsingEncoding:NSUTF8StringEncoding] options:NSJSONReadingMutableContainers error:nil];
        NSString * panLast4Digits = [additionalDataJson objectForKey:ADDITIONAL_DATA_PAN_LAST_4_DIGITS];
        NSString * cardImgFileName = [NSString stringWithFormat:@"%@%@.png", BASE_PATH , panLast4Digits];
        // Here calls loading card image and display
    }
}];
</code>
</pre><br/> <br/>

```ADDITIONAL_DATA_PAN_EXPIRY```
>   TBA


```MASTERCARD_CARD_LAYOUT_DESC```
>   Only for Mastercard.<br/>The key to get the card layout description which is the optional data element personalized by the issuer and containing the image of a card that for displaying on the MPA. This may not contain any value which depends on the Issuer.<br/> <br/>


```MASTERCARD_SECURITY_WORD```
>   Only for Mastercard.<br/> The key to get the MasterCard security word which is the static password used to authenticate the mobile MasterCard PayPass to the cardholder through the MPA.<br/> <br/>


```MASTERCARD_APPLICATION_LIFE_CYCLE_DATA```
>   Only for Mastercard.<br/> The key to get the application Life Cycle Data which contains the unique application ID of the issuer.<br/> <br/>


```MASTERCARD_MCHIP_ISSUER_OPTIONS```
>   Only for Mastercard.<br/> The key to get the M/Chip CVM Issuer Options which contains the issuer defined options related to the CVM applied by the application when working in the PayPass M/Chip mode.<br/> <br/>


```PAYMENT_CHANNELS```
>   Represents the field that identifies the supported payment channels for a card profile.<br/>
    The values are represented in hexadecimal format:
    <ul>
    <li>0x80 - CONTACTLESS</li>
    <li>0x20 - QR</li>
    <li>0xA0 - CONTACTLESS AND QR</li>
    </ul>
    <br/> <br/>

  • KnownMessageCode

    List of known message codes that are sent in a message from server.

    Constants with “REQUEST” prefix are sent in every server push that updates the card database on the device, to convey the operation to the UI application.

    Constants with “MSG” prefix are messages sent by server on some user’s operations that won’t get immediate feedback, such as NFC transaction or when the PIN has been changed from the issuer’s portal.

    Constants

    >  A digitized card was deleted with a WIPE_CARD command. <br/> <br/>

```REQUEST_SUSPEND_CARD```
>  A digitized card was suspended with a WIPE_PAYMENT_CREDS command. <br/> <br/>

```REQUEST_RESUME_CARD```
>  A digitized card was re-activated and replenished with a PROV_PAMENT_CREDS command. <br/> <br/>

```REQUEST_REPLENISH_KEYS```
>  A digitized card was replenished with a PROV_PAMENT_CREDS command. <br/> <br/>


```REQUEST_INSTALL_CARD```
>  Initial provisioning of personalization data was performed on a digitized card with a PROV_CARD. <br/> <br/>

```REQUEST_RENEW_CARD```
>  Renewal of personalization data was performed on a digitized card with a PROVE_CARD command. <br/> <br/>

```MSG_PAYMENT_INVALID_CRYPTOGRAM```
>  Payment can not be procedd due to wrong cryptorgram generated by SDK. <br/> <br/>

```MSG_PAYMENT_TRANSACTION_APPROVED```
>  Payment successful. <br/> <br/>

```MSG_PAYMENT_WRONG_PIN```

>  The end-user has entered a wrong PIN, for instance when making a payment transaction. This message has one parameter: the number of try remaining. <br/> <br/>

```MSG_PAYMENT_PIN_LOCKED```
>  Too many attemps to pay with wrong mobile pin. <br/> <br/>

```MSG_PIN_CHANGE_SUCCESSFUL```
>  Sent by the server when the PIN has been changed from another channel (not through SDK). <br/> <br/>
